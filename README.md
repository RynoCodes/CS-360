Briefly summarize the requirements and goals of the app you developed. What user needs was this app designed to address?
  The app was required to accept a weight input, save it to an SQL database and display the data in a grid format. The app is desinged to be a simple interface to track changing weight.
What screens and features were necessary to support user needs and produce a user-centered UI for the app? How did your UI designs keep users in mind? Why were your designs successful?
  The app needs four screens to function properly. The main page allows users to see their saved weights as well as acting as a hub to the other pages. The enter weight page allows a user to enter their daily weight, and the goal page allows a user to enter a custom goal. The text page allows a user to opt in to receiving text messages.
How did you approach the process of coding your app? What techniques or strategies did you use? How could those techniques or strategies be applied in the future?
  To code my app I started with the UI elements I would need on each screen. Once I had those I knew better what code to implement. I started with the most difficult thing first, the SQLDatabase. It was a struggle for me to get it to work, so starting with it allowed me to get the hardest part out of the way so the rest was downhill sailing.
How did you test to ensure your code was functional? Why is this process important, and what did it reveal?
  Testing is very important as it allows you to ensure your app works. To test it I mirrored the app on my personal device as I am not able to use the emulator. Everytime I tested the app I saw where it was breaking and had an idea of what I needed to do to fix it.
